# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest (`main`) | ✅ |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

To report a security issue, email the maintainer directly:

**karanbanjade@duck.com** (replace with your real contact)

Include as much detail as possible:
- A description of the vulnerability and its potential impact
- Steps to reproduce
- Any proof-of-concept code (if applicable)

You can expect a response within **72 hours**. If the issue is confirmed, a fix will be prioritized and a patched release published. You will be credited in the release notes unless you prefer to remain anonymous.

## Out of Scope

The following are **not** considered security vulnerabilities for this project:
- Issues only reproducible on unsupported/outdated dependencies
- Denial of service via resource exhaustion from a legitimate admin account
- Theoretical vulnerabilities without a practical attack scenario
